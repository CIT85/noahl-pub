New Super Mario Bros. DS Randomizer

What is it?

A Python script that randomizes the level locations in New Super Mario Bros. DS.

How does it work?

What is a Randomizer

A game randomizer shuffles the content in the game in an unpredictable manor. They're used to create a new experience from a familiar game that one has played before.

Randomizer's are fairly common, to the point where if a game exists then theres very likely a randomizer for it. Randomizer's are so popular, in fact, that an extended version known as archipelago has been created to randomize the contents of multiple games across each other. This means that you can find an item for one game inside of another, allowing for an extensive single player, and multiplayer experience.

How was this made?

This was made in Python using the library ndspy. It works by accessing each of the games levels and shuffling them about, randomizing the order in which you need to play each level. The levels in the game can be divided into areas, with there always containing one main area and up to two additional sub-areas.

The main crux of the randomizer are these areas, shuffling each area into a different area. This allows for more complex levels to be randomized as opposed to simply randomizing the main areas and ignoring sub-areas, however this randomization runs into it's own problems.

Known Issues and Future for the Project

Some known issues as well as what I want to do with the project in the future.

Known Issues

The most notable issue comes into play with how sub-levels are indexed. Each level is hardcoded with the number of areas that a level contains. In normal gameplay this is fine, but when trying to randomize the game there is a major issue when trying to shuffle a level with three areas into a level with two areas.

The ideal fix would be to update the area count when shuffling, however in decompiling the source code I have yet to find where this is coded into the base game. The second solution would be to just shuffle levels based on area count, so a level with two areas can only be swapped with another level with two areas. This is fine, but it does leave for less possibilities, and an overall less randomized/exciting experience for the user and is therefore not ideal.

Archipelago

I mentioned archipelago in the past, but only briefly. Archipelago is a randomizer for randomizers, allowing a games items to be shuffled into anothers. This project was actually made with Archipelago in mind, the idea was to finish the randomizer and then create an archipelago world, the format used for generating archipelago games, using levels and other in games items as checks. This goal is still attainable, and from working on an archipelago for another game, it's not super complicated, the main issue lies in the major known issue listed above.

Future Development

I'll likely continue working on this project in the future when I have more free time. I'll likely enlist help from people who have more experience working with the ndspy library than me. There are also a lot of quality of life things I want to add, including cleaning up UI, creating an executable so you don't have to run source code, as well as some in game changes like skipping the opening cutscene and displaying the correct level number when loading a level. At the very least I'll implement the less ideal fix for the sub-level indexing and create an open source repo for archipelago that other people can update on their own.
